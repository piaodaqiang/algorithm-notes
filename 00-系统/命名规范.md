# 命名规范

## 1. 总原则

文件名要满足三个目标：

- 一眼看懂内容
- 搜索时容易找到
- 长期引用时不容易变化

不要使用：

- `笔记1.md`
- `新建文档.md`
- `刷题.md`
- `重要.md`
- `未命名.md`

## 2. 推荐命名方式

### 算法专题

```text
二分查找.md
前缀和.md
01背包.md
最短路-Dijkstra.md
```

### STL 笔记

```text
vector.md
map.md
priority_queue.md
sort.md
lower_bound.md
```

### LeetCode 题解

```text
LC-0001-两数之和.md
LC-0206-反转链表.md
LC-0704-二分查找.md
```

### 错题复盘

```text
错题-LC-0704-二分边界错误.md
错题-考研机试-输入处理遗漏.md
```

### 模板

```text
模板-二分查找.md
模板-并查集.md
模板-Dijkstra.md
```

## 3. 中文还是英文

本仓库以中文学习和复习为主，因此：

- 概念类文件优先用中文。
- STL、函数名、类名保留英文原名。
- LeetCode 文件使用 `LC-编号-中文题名.md`。

这样既方便理解，也方便搜索英文 API。

## 4. Obsidian 双链命名

双链应尽量链接到稳定概念名：

```markdown
[[二分查找]]
[[vector]]
[[01背包]]
```

不要链接到太口语化的名字：

```markdown
[[那个二分的东西]]
[[今天错的题]]
```

链接名越稳定，知识网络越干净。
