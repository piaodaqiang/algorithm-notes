# 指针作为函数参数

## 学习主题

本篇整理“把变量地址传给函数”的写法。

对应案例：

- `pointer_to_array/ex1_pointer_to_array.cpp`：把 `par` 的地址传给 `getSeconds`，函数内部通过指针修改它。

## 核心结论

1. 如果函数参数是指针，可以把变量地址传进去。
2. `getSeconds(&par)` 中的 `&par` 表示把 `par` 的地址交给函数。
3. 函数内部用 `*par` 访问这个地址里的变量。
4. `*par = time(NULL);` 会修改外部变量 `par` 的值。

## 为什么会这样

普通参数传值时，函数拿到的是一份拷贝。

指针参数保存的是外部变量的地址。函数只要通过这个地址找到原来的变量，就可以修改它。

这个案例里，`main` 函数把 `par` 的地址传给 `getSeconds`，`getSeconds` 再通过 `*par` 给 `par` 赋值。

## 案例对应

`main` 函数中：

- 先定义 `unsigned long par;`
- 再调用 `getSeconds(&par);`
- 函数执行后，`par` 中保存了 `time(NULL)` 的结果。

`getSeconds` 函数中：

- 参数 `unsigned long* par` 是一个指针。
- `*par = time(NULL);` 表示给这个指针指向的位置赋值。

## 易错点

- 调用时要传地址，所以写 `&par`。
- 函数参数是 `unsigned long* par`，说明它接收的是地址。
- 函数内部赋值时要写 `*par = ...`，否则只是改指针变量本身。
- 使用指针参数前，要确认传入的地址是有效的。

## 关键代码摘录

下面不是完整源码，只保留和“指针作为函数参数”直接相关的片段。

### 函数声明

```cpp
void getSeconds(unsigned long* par);
```

参数 `par` 是指针，保存 `unsigned long` 变量的地址。

### 传入变量地址

```cpp
unsigned long par;

getSeconds(&par);
```

`&par` 表示取得 `par` 的地址。

### 通过指针修改变量

```cpp
void getSeconds(unsigned long* par)
{
    *par = time(NULL);
}
```

`*par` 表示指针指向的那个变量，所以这句会修改 `main` 函数里的 `par`。
