# 指针和数组

## 学习主题

本篇整理指针和数组的关系。

对应案例：

- `pointer_vs_array/ex1_pointer_vs_array.cpp`：用指针遍历数组。
- `pointer_to_array/ex2_pointer_to_array.cpp`：把数组传给函数求平均值。

## 核心结论

1. 数组名常常可以当作数组首元素地址使用。
2. `int* ptr = var;` 中，`ptr` 指向数组第一个元素。
3. `ptr++` 可以让指针移动到下一个数组元素。
4. 函数参数 `int *arr` 可以接收数组首元素地址。
5. `arr[i]` 和 `*(arr + i)` 都可以访问数组元素。

## 为什么会这样

数组是一段连续的空间。

当指针保存数组首元素地址后，就可以通过指针移动访问后面的元素。`ptr++` 会让指针从当前元素移动到下一个元素。

函数接收数组时，常见写法是接收数组首元素地址，再额外传入数组大小。

## 案例对应

`pointer_vs_array/ex1_pointer_vs_array.cpp` 中：

- `var` 是数组。
- `int* ptr = var;` 让指针指向数组开头。
- 循环中输出 `ptr` 和 `*ptr`，再通过 `ptr++` 移动到下一个元素。

`pointer_to_array/ex2_pointer_to_array.cpp` 中：

- `getAverage(arr, 5)` 把数组传入函数。
- 函数参数写成 `int *arr`。
- 函数内部用 `arr[i]` 累加数组元素。

## 易错点

- `ptr` 是地址，`*ptr` 才是当前元素的值。
- 用指针遍历数组时，要知道数组大小，否则容易访问越界。
- 函数参数中只有 `int *arr` 时，函数本身不知道数组有多长，所以还要传 `size`。
- `arr[i]` 看起来像数组写法，本质上仍然可以理解为通过地址偏移访问元素。

## 关键代码摘录

### 用指针遍历数组

```cpp
const int MAX = 3;
int var[MAX] = {10,20,30};
int* ptr = var;

for (int i = 0 ; i < MAX ; i++) {
    cout << ptr << endl;
    cout << *ptr << endl;

    ptr++;
}
```

重点看：

- `ptr` 输出当前元素地址。
- `*ptr` 输出当前元素值。
- `ptr++` 移动到下一个元素。

### 数组作为函数参数

```cpp
double getAverage(int *arr, int size);

int arr[] = {1000, 2, 3, 17, 50};
double average = getAverage(arr, 5);
```

`arr` 传进去的是数组首元素地址。

### 在函数中访问数组

```cpp
double getAverage(int *arr , int size)
{
    int sum = 0;

    for (int i = 0 ; i < size ; i ++) {
        sum += arr[i];
    }

    return static_cast<double>(sum) / size;
}
```

这里 `arr[i]` 可以理解为访问从 `arr` 开始的第 `i` 个元素。
