# 函数返回数组

## 学习主题

本篇整理 C++ 中“从函数返回数组”的基础写法。

在 C++ 里，函数不能直接把普通数组当作返回值返回。实际代码中通常返回==数组首元素的地址==，也就是返回一个指针，再通过这个指针访问数组内容。

本笔记对应两个案例：

- `ex1_getRandom.cpp`：使用 `static` 数组返回数组首地址。
- `ex2_createArray.cpp`：使用 `new[]` 动态数组返回数组首地址。
- `return_pointer_from_function/ex1_return_pointer_from_function.cpp`：再次练习用 `static` 数组从函数返回数组首地址。

## 核心结论

1. 普通数组不能直接作为函数返回值返回。
2. 不要返回函数内部普通局部数组的地址，因为函数结束后局部数组会被销毁。
3. 如果确实要让函数返回一段数组数据，可以使用 `static` 数组，或者使用 `new[]` 动态申请数组。

## 为什么不能直接返回局部数组

函数内部定义的普通局部数组，只在这个函数执行期间有效。

如果函数结束了，局部数组也就离开了自己的作用范围，它占用的空间不再可靠。此时如果把它的地址返回给外部，外部拿到的指针就可能指向一块已经无效的内存。

所以不要写类似这样的思路：

```cpp
int* func()
{
    int arr[10];
    return arr; // 错误思路：arr 是局部数组
}
```

## 方式一：返回 static 数组首地址

在 `ex1_getRandom.cpp` 中，函数 `getRandom()` 使用了 `static int r[10]`：

```cpp
int * getRandom()
{
    static int r[10];
    // 给 r 填入随机数
    return r;
}
```

这里 `r` 是 `static` 数组，不会随着 `getRandom()` 函数结束而销毁，会一直存在到程序结束。

函数返回 `r` 时，实际返回的是数组首元素地址，也就是 `&r[0]`。主函数中用 `int *p` 接收：

```cpp
int * p;
p = getRandom();
```

之后可以通过指针访问数组元素：

```cpp
*(p + i)
```

也可以写成更像数组的形式：

```cpp
p[i]
```

这个案例中还用到了随机数：

- `srand(static_cast<unsigned int>(time(nullptr)))` 用来设置随机数种子。
- ==`rand() % (10 - 2 + 1) + 2` 用来生成 `2` 到 `10` 之间的随机数。==
- `rand()` 不能写参数。

在 `return_pointer_from_function/ex1_return_pointer_from_function.cpp` 中，也使用了同样的思路：

```cpp
int * getRandom()
{
    static int r[10];
    return r;
}
```

这里的重点仍然是：`static int r[10]` 不会在函数结束后失效，`return r;` 返回的是数组首元素地址。

## 方式二：返回动态数组首地址

在 `ex2_createArray.cpp` 中，函数 `createArray(size)` 使用 `new[]` 在堆区申请数组：

```cpp
int * createArray(int size)
{
    int * arr = new int [size];

    for (int i = 0 ; i < size ; i++) {
        arr[i] = i + 1;
    }

    return arr;
}
```

这里 `arr` 保存的是动态数组首元素地址。函数结束后，`arr` 这个局部指针变量本身会消失，但它指向的堆区数组还在，所以可以把这个地址返回给主函数继续使用。

主函数中：

```cpp
int size = 10;
int * arr = createArray(size);
```

使用完动态数组后，必须释放内存：

```cpp
delete[] arr;
```

因为数组是用 `new[]` 申请的，所以释放时要用 `delete[]`，不能只写 `delete`。

## 两个案例的对应关系

| 案例文件 | 函数 | 返回方式 | 数组位置 | 谁负责释放 |
| --- | --- | --- | --- | --- |
| `ex1_getRandom.cpp` | `getRandom()` | 返回 `static int r[10]` 的首地址 | 静态存储区 | 不需要手动释放 |
| `ex2_createArray.cpp` | `createArray(size)` | 返回 `new int[size]` 的首地址 | 堆区 | 调用者用 `delete[]` 释放 |
| `return_pointer_from_function/ex1_return_pointer_from_function.cpp` | `getRandom()` | 返回 `static int r[10]` 的首地址 | 静态存储区 | 不需要手动释放 |

## 易错点

- 不要返回普通局部数组的地址。
- `return r;` 返回的是数组首元素地址，不是把整个数组复制出来。
- `int *p` 或 `int *arr` 保存的是地址，访问元素时可以用 `p[i]` 或 `*(p + i)`。
- `static` 数组生命周期很长，但不是每次调用函数都创建一份新的数组。
- `new[]` 申请的动态数组，用完后要配对 `delete[]`。
- `rand()` 不能带参数，随机数范围要通过取模和偏移来控制。

## 关键代码摘录

下面不是完整源码，只保留和“函数返回数组”直接相关的部分，复习时先看这些片段即可。

### `ex1_getRandom.cpp`：`static` 数组

```cpp
int * getRandom()
{
    static int r[10];

    for (int i = 0 ; i < 10 ; i ++) {
        r[i] = rand() % (10 - 2 + 1) + 2;
    }

    return r;
}
```

这个函数的重点是：

- 返回类型写成 `int *`，表示返回 `int` 类型数据的地址。
- `static int r[10]` 不会随着函数结束而销毁。
- `return r;` 返回的是 `r[0]` 的地址。

主函数中接收和访问：

```cpp
int * p;
p = getRandom();

for (int i = 0 ; i < 10 ; i ++) {
    cout << *(p + i) << endl;
}
```

这里的 `*(p + i)` 也可以写成 `p[i]`。

### `return_pointer_from_function/ex1_return_pointer_from_function.cpp`：从函数返回指针

```cpp
int * getRandom()
{
    static int r[10];

    for (int i = 0; i < 10 ; i++) {
        r[i] = rand() % 100 + 1;
    }

    return r;
}
```

这个新案例和前面的 `static` 数组思路相同：

- `static int r[10]` 避免函数结束后数组失效。
- `return r;` 返回数组首元素地址。
- 主函数中用 `int * p = getRandom();` 接收。

```cpp
int * p = getRandom();

for (int i = 0 ; i < 10 ; i++) {
    cout << p[i] << endl;
}
```

这里 `p[i]` 是通过返回的数组首地址访问数组元素。

### `ex2_createArray.cpp`：动态数组

```cpp
int * createArray(int size)
{
    int * arr = new int [size];

    for (int i = 0 ; i < size ; i++) {
        arr[i] = i + 1;
    }

    return arr;
}
```

这个函数的重点是：

- `new int[size]` 在堆区申请一段连续空间。
- `arr` 保存动态数组首元素地址。
- `return arr;` 把这段动态数组的首地址返回给调用者。

主函数中使用和释放：

```cpp
int size = 10;
int * arr = createArray(size);

for (int i = 0 ; i < size ; i++) {
    cout << arr[i] << endl;
}

delete[] arr;
```

只要用了 `new[]`，用完后就要记得配对 `delete[]`。
