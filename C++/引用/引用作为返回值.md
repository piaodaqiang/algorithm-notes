# 引用作为返回值

## 学习主题

本篇整理 C++ 中把引用作为函数返回值的写法。

对应案例：

- `ex3_reference.cpp`：返回数组元素的引用，并通过函数调用结果修改数组。
- `ex4_reference.cpp`：返回静态变量的引用，并通过引用修改静态变量。

## 核心结论

1. 函数返回类型可以写成引用，例如 `double&` 或 `int&`。
2. 如果函数返回的是某个变量的引用，函数调用结果可以放在赋值号左边。
3. `setValues(1) = 20.5;` 会修改 `vals[1]`。
4. 返回静态变量引用时，静态变量在函数结束后仍然有效。

## 为什么会这样

普通函数返回值通常是一份结果。

引用返回值返回的不是一份新数据，而是某个已有变量的别名。只要这个变量在函数结束后仍然有效，就可以通过返回的引用继续访问或修改它。

`ex3_reference.cpp` 中返回的是全局数组 `vals` 的某个元素引用。`ex4_reference.cpp` 中返回的是 `static int num` 的引用。

## 案例对应

`ex3_reference.cpp` 中：

- `double& setValues(int i)` 返回 `vals[i]` 的引用。
- `setValues(1) = 20.5;` 修改的是 `vals[1]`。
- `setValues(4) = 25.5;` 修改的是 `vals[4]`。

`ex4_reference.cpp` 中：

- `getStaticRef()` 返回静态变量 `num` 的引用。
- `int& ref = getStaticRef();` 让 `ref` 绑定到这个静态变量。
- `ref = 10;` 修改静态变量的值。

## 易错点

- 只有返回的对象在函数结束后==仍然有效==，返回引用才可靠。
- 案例中的 `vals` 是全局数组，函数结束后不会消失。
- 案例中的 `num` 是 `static` 变量，函数结束后也不会消失。
- 不要把函数内部普通局部变量的引用返回出去。
- `setValues(1) = 20.5;` 看起来像给函数赋值，实际是在给函数返回的引用赋值。

## 关键代码摘录

下面不是完整源码，只保留和“引用作为返回值”直接相关的片段。

### ex3_reference.cpp：返回数组元素的引用

```cpp
double vals[] = {10.1, 12.6, 33.1, 24.1, 50.0};

double& setValues(int i)
{
    double& ref = vals[i];
    return ref;
}
```

`setValues(i)` 返回的是 `vals[i]` 的引用。

### ex3_reference.cpp：通过返回引用修改数组元素

```cpp
setValues(1) = 20.5;
setValues(4) = 25.5;
```

这两句分别修改 `vals[1]` 和 `vals[4]`。

### ex4_reference.cpp：返回静态变量的引用

```cpp
int& getStaticRef()
{
    static int num = 5;
    return num;
}
```

`num` 是静态变量，函数结束后仍然存在。

### ex4_reference.cpp：通过引用修改静态变量

```cpp
int& ref = getStaticRef();

ref = 10;
```

`ref` 绑定到 `getStaticRef()` 返回的静态变量引用，所以修改 `ref` 会修改 `num`。
