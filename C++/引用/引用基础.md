# 引用基础

## 学习主题

本篇整理 C++ 引用的基础写法。

引用可以理解为变量的另一个名字。定义引用后，可以通过原变量名访问数据，也可以通过引用名访问同一份数据。

对应案例：

- `ex1_reference.cpp`：创建 `int` 引用和 `double` 引用，并通过引用输出变量的值。

## 核心结论

1. `int& r = i;` 表示 `r` 是变量 `i` 的引用。
2. 引用必须在定义时绑定到一个已经存在的变量。
3. 引用不是一份新的数据，而是原变量的别名。
4. 对原变量赋值后，通过引用名也能看到同一份值。

## 为什么会这样

普通变量名可以看成一块内存的名字。

引用相当于给这块内存再起一个名字。比如 `int& r = i;` 之后，`i` 和 `r` 都指向同一个变量内容。

所以案例里先给 `i` 赋值，再输出 `r`，得到的是同一个值。

## 案例对应

`ex1_reference.cpp` 中：

- `int& r = i;` 创建了 `i` 的引用。
- `double& s = d;` 创建了 `d` 的引用。
- 给 `i` 和 `d` 赋值后，输出 `r` 和 `s`，可以看到引用读到的是原变量的值。

## 易错点

- `&` 在 `int& r = i;` 中表示“引用声明”，不是取地址。
- 引用定义时==必须初始化==，不能只写一个空引用。
- 引用不是复制一份变量，对引用的理解重点是“别名”。
- 引用使用时直接写名字，不需要像指针那样写 `*` 解引用。

## 关键代码摘录

下面不是完整源码，只保留和“引用基础”直接相关的片段。

### ex1_reference.cpp：创建 int 和 double 引用

```cpp
int i;
double d;

int& r = i;
double& s = d;
```

这里 `r` 是 `i` 的引用，`s` 是 `d` 的引用。

### ex1_reference.cpp：通过引用读到原变量的值

```cpp
i = 5;
cout << i << endl;
cout << r << endl;

d = 11.7;
cout << d << endl;
cout << s << endl;
```

`i` 和 `r` 对应同一份 `int` 数据，`d` 和 `s` 对应同一份 `double` 数据。
