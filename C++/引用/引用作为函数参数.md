# 引用作为函数参数

## 学习主题

本篇整理 C++ 中把引用作为函数参数的写法。

对应案例：

- `ex2_reference.cpp`：用 `swap(int& x, int& y)` 交换两个变量的值。

## 核心结论

1. 函数参数写成 `int& x`，表示 `x` 是传入变量的引用。
2. 调用引用参数函数时，直接写变量名，例如 `swap(a, b)`。
3. 函数内部修改 `x` 和 `y`，会直接影响外面的 `a` 和 `b`。
4. 引用参数适合让函数修改调用者传进来的变量。

## 为什么会这样

如果函数参数是普通 `int x`，函数拿到的是外部变量的一份拷贝，修改 `x` 不会影响外面的变量。

如果函数参数是 `int& x`，`x` 就是外部变量的别名。函数里改 `x`，本质上就是在改外面的那个变量。

所以 `swap(a, b)` 调用之后，`main` 函数里的 `a` 和 `b` 会真正交换。

## 案例对应

`ex2_reference.cpp` 中：

- 函数声明为 `void swap(int& x,int& y);`
- 主函数中直接调用 `swap(a,b);`
- `swap` 函数内部通过 `temp` 完成交换。

因为 `x` 和 `y` 是 `a`、`b` 的引用，所以交换后外部变量也会改变。

## 易错点

- 调用引用参数函数时写 `swap(a, b)`，不是 `swap(&a, &b)`。
- `int& x` 不是普通参数，函数内部修改它会影响外部变量。
- 引用参数语法比指针参数更像普通变量，但效果上仍然可以修改外部对象。
- 写交换逻辑时需要临时变量 `temp`，否则一个值可能会被覆盖。

## 关键代码摘录

下面不是完整源码，只保留和“引用作为函数参数”直接相关的片段。

### ex2_reference.cpp：声明引用参数函数

```cpp
void swap(int& x,int& y);
```

`x` 和 `y` 都是引用参数。

### ex2_reference.cpp：调用时直接传变量名

```cpp
int a = 100;
int b = 200;

swap(a,b);
```

这里不需要写 `&a` 或 `&b`。

### ex2_reference.cpp：用引用参数交换两个变量

```cpp
void swap(int& x, int& y)
{
    int temp;
    temp = x;
    x = y;
    y = temp;
}
```

`x` 和 `y` 是外部变量的别名，所以这里修改 `x`、`y` 会影响 `a`、`b`。
