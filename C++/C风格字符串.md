# C风格字符串

## 学习主题

本篇整理 C++ 中的 C 风格字符串。

这次两个案例主要学习的是：

- 用 `char` 数组保存字符串。
- 字符串末尾需要有结束标记 `'\0'`。
- 使用 `<cstring>` 里的 `strcpy`、`strcat`、`strlen` 处理字符串。

注意：这里整理的是 C 风格字符串，不是后面会学到的 `std::string`。

## 核心结论

1. C 风格字符串本质上是一个 `char` 数组。
2. 字符串必须以 `'\0'` 结尾，否则输出时不知道在哪里停止。
3. 使用 `strcpy`、`strcat` 这类函数时，目标数组空间必须足够。

## 字符数组和字符串的关系

普通字符数组只是连续存放多个字符。

如果这个字符数组最后有 `'\0'`，它就可以当作 C 风格字符串使用。

例如：

```cpp
char site[13] = {'H','U','A','N','G','Q','I','N','G','T','A','N','\0'};
```

这里前 12 个位置存放字符，最后一个位置存放 `'\0'`。

如果用 `cout` 输出这个数组：

```cpp
cout << site << endl;
```

程序会从数组第一个字符开始输出，直到遇到 `'\0'` 才停止。

## 为什么末尾需要 `'\0'`

C 风格字符串没有单独保存“字符串长度”。

所以程序需要靠 `'\0'` ==判断字符串在哪里结束==。

如果没有 `'\0'`，程序可能会继续往后读内存，直到碰巧遇到一个 `'\0'`，这会导致输出乱码或其他不可预期的问题。

## `ex1.cpp`：手动用字符数组保存字符串

案例中的核心代码是：

```cpp
char site[13] = {'H','U','A','N','G','Q','I','N','G','T','A','N','\0'};

cout << "site[13]: " << site << endl;
```

这个例子说明：

- `site` 是一个 `char` 数组。
- 数组大小是 `13`，其中 `12` 个位置放字符，`1` 个位置放 `'\0'`。
- 输出 `site` 时，不会输出数组长度，而是输出从开头到 `'\0'` 之前的内容。

## `ex2_string_function.cpp`：字符串处理函数

这个案例使用了：

```cpp
#include <cstring>
```

因为 `strcpy`、`strcat`、`strlen` 来自 `<cstring>`。

### `strcpy`：复制字符串

```cpp
char str1[13] = "huangqingtan";
char str3[13];

strcpy(str3, str1);
```

含义：把 `str1` 中的字符串复制到 `str3` 中。

这里 `str3[13]` 可以装下 `"huangqingtan"`，因为：

- `"huangqingtan"` 有 12 个普通字符。
- 末尾还需要 1 个 `'\0'`。
- 总共需要 13 个 `char` 空间。

### `strcat`：连接字符串

```cpp
char str1[13] = "huangqingtan";
char str2[13] = "helloworld";

strcat(str1, str2);
```

含义：把 `str2` 拼接到 `str1` 后面。

但这个案例里要特别注意：`str1[13]` 的空间只刚好够放 `"huangqingtan"` 和末尾的 `'\0'`。

执行 `strcat(str1, str2)` 后，`str1` 需要同时容纳：

- `"huangqingtan"`：12 个字符。
- `"helloworld"`：10 个字符。
- 结尾 `'\0'`：1 个字符。

总共需要 23 个 `char` 空间，但 `str1` 只有 13 个空间，所以这里存在==数组越界风险==。

## `strlen`：计算字符串长度

```cpp
len = strlen(str1);
```

`strlen` 用来计算字符串长度，但它计算的是 `'\0'` 之前的字符个数，==不包含最后的 `'\0'`==。

例如 `"huangqingtan"` 的长度是 `12`，但保存它至少需要 `13` 个 `char` 空间。

## 重要易错点

- 字符串末尾需要 `'\0'`，数组==容量==要把这个位置算进去。
- `"huangqingtan"` 看起来是 12 个字符，但作为 C 风格字符串保存时需要 13 个空间。
- `strlen` 返回的是字符数量，不包含 `'\0'`。
- `strcpy` 会把源字符串复制到目标数组里，目标数组必须足够大。
- `strcat` 会把第二个字符串拼到第一个字符串后面，第一个数组必须有足够的剩余空间。
- 本次 `ex2_string_function.cpp` 中的 `strcat(str1, str2)` 有越界风险，因为 `str1[13]` 容量不够。
