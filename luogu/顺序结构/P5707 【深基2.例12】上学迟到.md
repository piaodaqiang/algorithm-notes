# P5707 【深基2.例12】上学迟到

## 题型
顺序结构 / 时间换算 / 向上取整 / 格式化输出

## 本题练习的 C++ 知识
- `ceil()` 和 `<cmath>`：把走路时间向上取整。
- `static_cast<double>(s)`：把整数转成 `double`，避免整数除法。
- `setw(2)`、`setfill('0')` 和 `<iomanip>`：控制 `HH:MM` 的两位补零输出。
- 分钟和 `HH:MM` 之间的转换。

## 解题思路
这题可以把所有时间都先换算成“分钟”来处理，这样会比直接处理小时和分钟简单很多。

第一步，计算走路需要多少分钟。因为 yyy 只能在每分钟的第 0 秒出发，所以只要路上时间不是整数分钟，就必须多提前 1 分钟：

```cpp
int walk_time = ceil(static_cast<double>(s) / v);
```

第二步，加上垃圾分类需要的 10 分钟：

```cpp
int total_time = walk_time + 10;
```

第三步，把早上 8:00 转成分钟。一天从 00:00 开始算，8:00 就是：

```cpp
int arrival_time = 8 * 60;
```

第四步，用到达时间减去路上总耗时，得到最晚出发时间：

```cpp
int start_time = arrival_time - total_time;
```

如果 `start_time < 0`，说明要在前一天出发，需要补回一天的分钟数 `24 * 60`：

```cpp
while (start_time < 0) {
    start_time += 24 * 60;
}
```

最后，把总分钟数转回小时和分钟：

```cpp
int HH = start_time / 60;
int MM = start_time % 60;
```

## 易错点
- `start_time = + 24 * 60` 是赋值，不是累加。它会把 `start_time` 直接变成 `1440`，不是在原来的负数基础上加一天。
- 正确写法应该是 `start_time += 24 * 60`，更稳的写法是用 `while` 一直加到非负。
- `static_cast<double>(s) / v` 是为了先得到小数结果，再交给 `ceil()` 向上取整；如果直接 `s / v`，两个都是 `int`，会先做整数除法。
- `setw(2)` 只影响紧跟着输出的下一个数据，所以小时和分钟前面都要各写一次。
- `setfill('0')` 设置补位字符为字符 `'0'`，要包含 `<iomanip>` 才能使用 `setw` 和 `setfill`。

## 关键代码
```cpp
int walk_time = ceil(static_cast<double>(s) / v);
int total_time = walk_time + 10;
int start_time = 8 * 60 - total_time;

while (start_time < 0) {
    start_time += 24 * 60;
}

int HH = start_time / 60;
int MM = start_time % 60;

cout << setw(2) << setfill('0') << HH << ":"
     << setw(2) << setfill('0') << MM << endl;
```

## 复盘
以后遇到时间题，优先把时间统一换成“分钟”或“秒”来算，最后再转回题目要求的格式。

这题最关键的两个点是：路上时间要向上取整，跨到前一天时要加回 `24 * 60` 分钟。
