# P5709 【深基2.习6】Apples Prologue / 苹果和虫子

## 题型
分支结构 / 向上取整 / 边界处理 / 简单模拟

## 本题练习的 C++ 知识
- `if (t == 0)` 处理特殊边界，避免除以 0。
- `%` 判断是否整除。
- 整数除法 `s / t` 会自动向下取整。
- `max(a, 0)` 可以保证剩余苹果数不会小于 0；标准写法需要 `#include <algorithm>`。

## 解题思路
这题要求的是“还剩几个完整的苹果”。可以先算出已经被吃掉多少个苹果，再用总数 `m` 减去吃掉的数量。

最重要的特殊情况是 `t == 0`。如果吃一个苹果需要 0 分钟，那么在经过 `s` 分钟后苹果会被吃完；同时后面的 `s / t` 会出现除以 0，导致 RE。所以这里要提前输出 `0` 并结束程序：

```cpp
if (t == 0) {
    cout << 0 << endl;
    return 0;
}
```

当 `t > 0` 时，需要计算 `s` 分钟内吃掉了多少个苹果。因为“吃完一个后立刻开始吃下一个”，只要已经开始吃某个苹果，它就不是完整的苹果了，所以这里要向上取整。

你的写法是用 `s % t` 分情况处理：

```text
如果 s % t == 0，说明刚好吃完整数个苹果，吃掉 s / t 个。
如果 s % t != 0，说明还正在吃下一个苹果，完整苹果还要再少 1 个，所以吃掉 s / t + 1 个。
```

最后剩余数量不能小于 0，所以用 `max(..., 0)` 兜底：

```cpp
num = max(m - eaten, 0);
```

## 我容易错的地方
- 忘记处理 `t == 0`，后面写 `s / t` 或 `s % t` 就会除以 0，可能 RE。
- 注释里应该写“0 分钟”，不是“0 秒”，因为题目单位是分钟。
- `s / t` 是整数除法，结果会向下取整；如果有余数，还要额外算正在被吃的那个苹果。
- `max` 最稳妥要加 `#include <algorithm>`，不要依赖编译器间接包含。
- 剩余苹果数最少是 `0`，不要输出负数。

## 关键代码
```cpp
if (t == 0) {
    cout << 0 << endl;
    return 0;
}

if (s % t == 0) {
    num = max(m - s / t, 0);
} else {
    num = max(m - s / t - 1, 0);
}
```

## 复盘
以后看到“剩余完整物品数量”的题，要注意：正在被消耗但还没消耗完的那个，也不能算完整。

这题的关键是三个边界：`t == 0` 防止除以 0，`s % t != 0` 时要多算一个被吃掉的苹果，剩余数量不能低于 `0`。

以后也可以用下面这种整数公式计算向上取整，不过这题先理解你的分支写法更重要：

```cpp
int eaten = (s + t - 1) / t;
```
