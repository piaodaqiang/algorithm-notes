# P4414 [COCI 2006/2007 #2] ABC

## 题型
分支结构 / 排序 / 字符映射 / 输出格式

## 本题练习的 C++ 知识
- `sort(num, num + 3)` 可以把三个数从小到大排序。
- `string order` 可以保存没有空格的一串字符，例如 `ABC`、`CAB`。
- 字符可以参与减法，例如 `order[i] - 'A'`。
- 输出多个数据时，可以只在数字之间输出空格。

## 解题思路
题目给出的三个数不一定有序，但它们本质上对应 `A < B < C`。

所以第一步先排序：

```cpp
sort(num, num + 3);
```

排序后：

```text
num[0] 对应 A
num[1] 对应 B
num[2] 对应 C
```

然后读取输出顺序 `order`，按照 `order[i]` 输出对应的数。

## 两种写法对比

### 写法一：if / else if

```cpp
if (order[i] == 'A') cout << num[0];
else if (order[i] == 'B') cout << num[1];
else if (order[i] == 'C') cout << num[2];
```

这种写法直观，适合刚开始理解题意：看到 `A` 就输出最小值，看到 `B` 就输出中间值，看到 `C` 就输出最大值。

缺点是代码比较长，如果映射关系更多，会写很多判断。

### 写法二：字符转下标

```cpp
cout << num[order[i] - 'A'];
```

这里利用了字符编码连续的特点：

```text
'A' - 'A' = 0 -> num[0]
'B' - 'A' = 1 -> num[1]
'C' - 'A' = 2 -> num[2]
```

所以 `order[i] - 'A'` 刚好可以变成数组下标。这个写法更简洁，但前提是要理解字符也可以做减法。

## 空格输出

输出三个数时，中间要有空格，但最后一个数后面不需要多输出空格。可以这样写：

```cpp
if (i != 2) cout << " ";
```

意思是：只要当前不是最后一个位置，就在后面补一个空格。这是常见的“间隙输出空格”写法。

## 我容易错的地方
- 文件名里不能直接用题目中的 `/`，但正文标题可以保留。
- `order[i] - 'A'` 得到的是下标，不是字符本身。
- 字符转下标能成立，是因为 `A`、`B`、`C` 在字符编码里是连续的。
- 最后一个数字后面不要再额外输出空格。
- `#include <cstring>` 不是使用 `string` 的标准头文件，更推荐写 `#include <string>`。

## 关键代码
```cpp
sort(num, num + 3);

for (int i = 0; i < 3; i++) {
    cout << num[order[i] - 'A'];
    if (i != 2) cout << " ";
}
```

## 复盘
以后遇到类似 `A/B/C`、`0/1/2` 这种连续映射，可以想一想能不能用字符减法转成数组下标。

这题的核心收获是：先排序建立 `A/B/C` 和数组下标的对应关系，再用字符映射输出。
