# P1909 [NOIP 2016 普及组] 买铅笔

## 题型
分支结构 / 向上取整 / 枚举三种方案 / 最小值

## 本题练习的 C++ 知识
- `ceil()` 来自 `<cmath>`，作用是向上取整。
- `1.0 * n` 可以把后面的除法变成浮点数除法。
- `static_cast<double>(n) / a[i]` 也可以，而且更明确。
- `sort(money, money + 3)` 来自 `<algorithm>`，排序区间是 `[money, money + 3)`。

## 解题思路
这题有 3 种包装。因为商店不允许拆开包装，所以每一种包装都要单独计算：如果只买这种包装，至少要买多少包，最后要花多少钱。

假设一种包装里有 `a[i]` 支铅笔，价格是 `b[i]`，需要买够至少 `n` 支。需要买的包数是：

```cpp
ceil(1.0 * n / a[i])
```

这里必须向上取整。比如需要 `57` 支，一包 `30` 支：

```text
57 / 30 = 1.9
```

实际不能买 `1.9` 包，只能买 `2` 包，所以要用 `ceil()` 向上取整。

每种包装的总花费就是：

```cpp
money[i] = ceil(1.0 * n / a[i]) * b[i];
```

算出三种包装的花费后，可以排序，然后输出最小的：

```cpp
sort(money, money + 3);
cout << money[0] << endl;
```

也可以不排序，边算边维护最小值。这题只有 3 种包装，用排序写起来也很直观。

## 关于 `1.0 * n` 和 `static_cast<double>(n)`

不能直接写：

```cpp
ceil(n / a[i])
```

原因是 `n` 和 `a[i]` 都是 `int`，所以 `n / a[i]` 会先做整数除法。比如：

```text
57 / 30 = 1
```

小数部分已经被丢掉了，再传给 `ceil()` 也只能得到 `1`，这就错了。

写成：

```cpp
ceil(1.0 * n / a[i])
```

是因为 `1.0` 是 `double`，`1.0 * n` 会让整个表达式变成浮点数计算：

```text
1.0 * 57 / 30 = 1.9
ceil(1.9) = 2
```

也可以写成：

```cpp
ceil(static_cast<double>(n) / a[i])
```

这两种写法都能过。区别是：

```text
1.0 * n：写起来短，竞赛里常见。
static_cast<double>(n)：更明确地表达“我要把 n 转成 double”，更像现代 C++ 的写法。
```

所以不是不能用 `static_cast<double>(n)`，而是两种都可以。理解之后，平时学习更推荐逐渐习惯 `static_cast<double>(n)`，因为它的意图更清楚。

## 我容易错的地方
- `ceil(n / a[i])` 是错的，因为整数除法会先丢掉小数部分。
- 不能只看“每支铅笔单价最低”，因为必须整包购买，可能会多买很多支。
- `ceil()` 返回的是浮点类型，这题赋给 `int` 可以用，但要知道它本身不是整数函数。
- `sort(money, money + 3)` 的右边界是最后一个元素后面的位置，不是 `money + 2`。
- 这题也可以用整数公式计算包数，避免浮点数：`(n + a[i] - 1) / a[i]`。

## 关键代码
```cpp
for (int i = 0; i < 3; i++) {
    cin >> a[i] >> b[i];
    money[i] = ceil(1.0 * n / a[i]) * b[i];
}

sort(money, money + 3);
cout << money[0] << endl;
```

也可以把包数单独写出来，更容易读：

```cpp
int cnt = ceil(static_cast<double>(n) / a[i]);
int cost = cnt * b[i];
```

如果不用 `ceil()`，可以用整数向上取整公式：

```cpp
int cnt = (n + a[i] - 1) / a[i];
```
