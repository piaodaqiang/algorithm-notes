# P5717 【深基3.习8】三角形分类

## 题型
分支结构 / 排序 / 三角形判定 / 分类输出

## 本题练习的 C++ 知识
- `sort(a, a + 3)` 来自 `<algorithm>`，可以把长度为 3 的数组从小到大排序。
- 排序后 `a[0] <= a[1] <= a[2]`，可以把它们理解成短边、次短边、最长边。
- 直角、锐角、钝角可以比较 `a[0] * a[0] + a[1] * a[1]` 和 `a[2] * a[2]`。

## 解题思路
这题先把三条边放进数组并排序：

```cpp
sort(a, a + 3);
```

排序后，`a[2]` 一定是最长边。判断三条边能不能组成三角形，只需要看最短的两条边之和是否大于最长边：

```cpp
if (a[0] + a[1] <= a[2]) {
    cout << "Not triangle";
    return 0;
}
```

所以：如果已经排序，第一个 `if` 可以只写 `a[0] + a[1] <= a[2]`。

如果没有排序，就不能只判断这一组。因为你不知道哪条边最长，需要检查三组边和关系，或者先找出最长边。

## 数组和普通变量的取舍

数组写法的优点：

```text
可以直接 sort。
适合这种“不关心原来哪条边叫 a/b/c，只关心大小顺序”的题。
排序后判断更统一。
```

数组写法的缺点：

```text
a[0]、a[1]、a[2] 的可读性不如 a、b、c。
必须记住排序后 a[0] 是短边，a[2] 是最长边。
```

普通变量 `a, b, c` 的优点是直观；缺点是如果想按大小判断，要自己写比较交换，或者多写几组判断，代码更容易乱。

## 我容易错的地方
- 排序后可以只判断 `a[0] + a[1] <= a[2]`；没排序不能只这样写。
- 不能组成三角形时要立刻 `return 0`，不要继续输出其他类型。
- 等边三角形也属于等腰三角形，所以要先输出 `Isosceles triangle`，再输出 `Equilateral triangle`。
- `sort(a, a + 3)` 的右边界是最后一个元素后面的位置，不是最后一个元素本身。

## 关键代码
```cpp
sort(a, a + 3);

if (a[0] + a[1] <= a[2]) {
    cout << "Not triangle";
    return 0;
}

if (a[0] * a[0] + a[1] * a[1] == a[2] * a[2]) cout << "Right triangle" << endl;
if (a[0] * a[0] + a[1] * a[1] > a[2] * a[2]) cout << "Acute triangle" << endl;
if (a[0] * a[0] + a[1] * a[1] < a[2] * a[2]) cout << "Obtuse triangle" << endl;
```

## 复盘
这题的关键经验是：当题目和“大小关系”有关时，先排序可以大幅减少判断。

排序后，问题会变成：只看最短边、次短边、最长边之间的关系。这样比直接用三个变量硬判断更清楚。

### 完整代码
```cpp
#include<iostream>
#include<algorithm>
using namespace std;

int main()
{
    int a[3];
    cin >> a[0] >> a[1] >> a[2];
    sort(a,a+3);
    if(a[0]+a[1]<=a[2] || a[0]+a[2]<=a[1] || a[1]+a[2]<=a[0]) {cout << "Not triangle"; return 0;} 
    if(a[0]*a[0]+a[1]*a[1]==a[2]*a[2]) cout << "Right triangle" << endl;
    if(a[0]*a[0]+a[1]*a[1]>a[2]*a[2]) cout << "Acute triangle" << endl;
    if(a[0]*a[0]+a[1]*a[1]<a[2]*a[2]) cout << "Obtuse triangle" << endl;
    if(a[0]==a[1] || a[0]==a[2] || a[1]==a[2]) cout << "Isosceles triangle" << endl;
    if(a[0]==a[1] && a[0]==a[2] && a[1]==a[2]) cout << "Equilateral triangle" << endl;
    return 0;
}
```
