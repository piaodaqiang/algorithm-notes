# P5716 【深基3.例9】月份天数

## 题型
分支结构 / 闰年判断 / 数组查表

## 本题练习的 C++ 知识
- 一维数组初始化：`int month[13] = {...};`
- 用 `month[m]` 根据月份直接取天数。
- 闰年判断：`y % 400 == 0 || (y % 4 == 0 && y % 100 != 0)`。
- `&&` 的优先级高于 `||`，但建议加括号让逻辑更清楚。

## 解题思路
这题要根据年份和月份输出当月天数。普通年份每个月的天数是固定的，所以可以先用数组把它们存起来：

```cpp
int month[13] = {0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
```

这里数组开到 `13`，是为了让下标 `1` 到 `12` 正好对应一月到十二月。`month[0]` 不代表任何月份，只是占位。

然后判断这一年是不是闰年。如果是闰年，就把二月改成 `29` 天：

```cpp
if (y % 400 == 0 || (y % 4 == 0 && y % 100 != 0)) {
    month[2] = 29;
}
```

最后直接输出：

```cpp
cout << month[m];
```

这个思路叫“查表”。当题目里有一组固定对应关系时，可以先把表存起来，需要哪个就按下标取哪个。

## switch 写法和数组写法对比

### switch 写法

`switch` 的优点是直观，不需要先理解数组下标。比如 `case 1`、`case 3`、`case 5` 输出 `31`，`case 4`、`case 6` 输出 `30`。

但它的缺点也明显：分支比较多，代码容易变长；如果以后要调整规则，需要在多个分支里改，维护起来不如数组方便。

### 数组查表写法

数组法的优点是简洁。月份和天数是一一对应的，用 `month[m]` 就能直接得到答案。

这题数据规模很小，`switch` 和数组在真实运行时间上差异不是重点。更重要的是：数组查表更适合“固定映射关系”，代码可读性和维护性更好。

数组法需要注意的是，要理解为什么 `month[0]` 用来占位，以及为什么 `month[2]` 表示二月。

## 我容易错的地方
- 二月默认是 `28`，只有闰年才改成 `29`。
- `month[13]` 的有效下标是 `0` 到 `12`，不能访问 `month[13]`。
- 闰年条件建议写成 `y % 400 == 0 || (y % 4 == 0 && y % 100 != 0)`，括号能减少读错的可能。
- 这题可以把数组写成全局变量，也可以写在 `main` 里面；全局数组会被整个程序共享。
- 不要把“数组法更简洁”简单理解成“它一定运行时间快很多”，这题核心是代码结构更清楚。

## 关键代码
```cpp
int month[13] = {0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};

if (y % 400 == 0 || (y % 4 == 0 && y % 100 != 0)) {
    month[2] = 29;
}

cout << month[m];
```

## 复盘
以后遇到“某个编号对应某个固定值”的题，可以优先想数组查表。

这题给我的收获是：有些分支结构题不一定非要写很多 `switch` 或 `if`。当数据本身像一张固定表时，先把表存起来，再通过下标查询，往往更简洁。
