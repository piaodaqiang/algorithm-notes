# P5710 【深基3.例2】数的性质

## 题型
分支结构 / 布尔逻辑 / 条件表达式

## 本题练习的 C++ 知识
- `bool` 可以保存一个性质是否成立。
- `&&` 表示两个条件同时成立。
- `||` 表示至少一个条件成立。
- `!=` 可以用来表示“两个布尔值不同”，也就是刚好一个成立。
- `!p1 && !p2` 表示两个性质都不成立。

## 解题思路
这题一开始很容易想到用很多 `if` 分别判断四个人喜不喜欢这个数。但看了题解后可以发现，这题更好的思路是先把两个基础性质拆出来。

性质 1 是“是偶数”：

```cpp
bool p1 = x % 2 == 0;
```

性质 2 是“大于 4 且不大于 12”：

```cpp
bool p2 = x > 4 && x <= 12;
```

接下来四个人的喜好其实就是四种逻辑关系：

```text
小 A：两个性质同时成立        -> p1 && p2
Uim：至少符合其中一个性质      -> p1 || p2
小 B：刚好符合其中一个性质      -> p1 != p2
正妹：两个性质都不符合          -> !p1 && !p2
```

所以这题不一定要写很多层条件判断。先把题目中的“性质”变成布尔变量，再用逻辑表达式组合，代码会更短，也更贴近题目本身。

## 我容易错的地方
- 容易把每个人的情况都写成一大段 `if`，导致代码重复又不清晰。
- “刚好符合一个性质”可以用 `p1 != p2`，因为两个布尔值一真一假时才不同。
- 输出顺序不能乱，必须是小 A、Uim、小 B、正妹。
- 每个结果之间要输出空格。
- 这题的重点不是新语法，而是把复杂判断抽象成简单的逻辑表达式。

## 关键代码
```cpp
bool p1 = x % 2 == 0;
bool p2 = x > 4 && x <= 12;

cout << (p1 && p2) << " "
     << (p1 || p2) << " "
     << (p1 != p2) << " "
     << (!p1 && !p2) << endl;
```

## 复盘
以后遇到这种“先给几个性质，再问不同人/不同情况是否满足”的题，可以先不要急着写 `if`。

更好的做法是：

```text
先把基础性质保存成 bool 变量，
再根据题意用 &&、||、!、!= 组合这些变量。
```

这道题给我的收获是：有些分支结构题不一定真的需要写很多分支，布尔表达式本身就可以表达判断逻辑。
