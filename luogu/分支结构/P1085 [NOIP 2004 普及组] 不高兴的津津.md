# P1085 [NOIP 2004 普及组] 不高兴的津津

## 题型
分支结构 / 循环输入 / 最大值维护 / 模拟

## 本题练习的 C++ 知识
- `for (int i = 1; i <= 7; i++)` 可以对应周一到周日。
- 边输入边处理，不一定要把所有数据存进数组。
- 用两个变量维护当前最大值和最大值所在位置。
- 只在严格更大时更新，能自然保留最靠前的那一天。

## 解题思路
这题问的是：如果津津会不高兴，哪一天最不高兴。

我一开始容易想到把 7 天的数据都存进数组，再用多个循环找最大值、最小值。但这题其实不需要保存全部数据，因为最后只关心“目前为止最不高兴的是哪一天”。

题解的核心思路是：边读入，边维护当前答案。

可以用两个变量：

```cpp
int max_time = 8;
int max_day = 0;
```

`max_time` 表示目前记录到的最大上课时间。初始值设为 `8` 很巧妙，因为题目说超过 8 小时才会不高兴。如果 7 天都不超过 8 小时，就不会更新答案，最后 `max_day` 仍然是 `0`。

`max_day` 表示目前最不高兴的是周几。初始值是 `0`，表示还没有发现不高兴的日期。

每读入一天，就计算这一天总上课时间：

```cpp
int total = t1 + t2;
```

如果这一天比当前记录的 `max_time` 更大，就更新最大值和日期：

```cpp
if (total > max_time) {
    max_time = total;
    max_day = i;
}
```

这里必须用 `>`，不要用 `>=`。因为题目要求如果有多天不高兴程度相同，输出时间最靠前的一天。如果用 `>=`，后面的同样最大值会覆盖前面的日期。

## 我容易错的地方
- 不需要用数组保存 7 天数据，除非后面还要反复访问这些数据。
- 不需要多个循环找最大值、最小值，这题只需要维护当前最大值。
- 相同最大值时不能更新日期，否则会输出靠后的那天。
- 初始最大值设为 `8`，可以自然处理“没有一天超过 8 小时”的情况。
- 变量名 `max` 虽然能编译，但容易和标准库里的 `max` 混淆，以后更推荐写成 `max_time`。

## 关键代码
```cpp
int max_time = 8;
int max_day = 0;

for (int i = 1; i <= 7; i++) {
    int t1, t2;
    cin >> t1 >> t2;

    int total = t1 + t2;
    if (total > max_time) {
        max_time = total;
        max_day = i;
    }
}

cout << max_day;
```

## 复盘
这题给我的重要经验是：不是所有题都要先把数据全部存起来。

如果题目只问“最大值是多少”或者“最大值出现在哪里”，就可以边输入边更新答案。这样代码更短，也更不容易被数组和多个循环绕晕。

以后看到类似问题，可以先问自己：

```text
我真的需要保存所有数据吗？
还是只需要维护当前最优答案？
```

这类思路叫“边读入边维护答案”，在入门题里非常常见。
