# P2669 [NOIP 2015 普及组] 金币

## 题型
循环结构 / 分组模拟 / 嵌套循环 / 状态维护 / 分组求和

## 本题练习的 C++ 知识
- 用嵌套循环模拟分组规律。
- 用 `dg` 和 `di` 维护当前金币组和组内天数。
- 用 `i * i` 直接计算完整一组的金币数。
- 用变量 `day` 记录已经统计了多少天。
- 用变量 `coins` 累加金币总数。
- 理解 `return 0` 和 `break` 的区别。

## 解题思路
这题可以想象成一个“阶梯表”：

```text
第 1 组：1 天，每天 1 枚金币
第 2 组：2 天，每天 2 枚金币
第 3 组：3 天，每天 3 枚金币
第 4 组：4 天，每天 4 枚金币
...
```

外层循环的 `i` 表示当前这一组每天发多少枚金币，也表示这一组一共有多少天。

内层循环的 `j` 表示当前组内第几天。每模拟一天，就把当天金币数加到 `coins`，同时让 `day++`。

当 `day == k` 时，说明前 `k` 天已经全部统计完了，此时直接输出答案并结束程序。

## 我容易错的地方
- `break` 只能跳出当前所在的一层循环。如果在内层循环里写 `break`，只能跳出内层，外层循环还会继续。
- 这里想要统计完第 `k` 天后直接结束整个程序，所以用 `return 0` 更合适。
- `day++` 要和“过了一天”同步，每模拟一天就加一次。
- 判断 `day == k` 要放在 `coins += i` 和 `day++` 之后，因为第 `k` 天的金币也要算进去。

## 关键代码
```cpp
for (int i = 1; i <= k; i++) {
    for (int j = 1; j <= i; j++) {
        coins += i;
        day++;

        if (day == k) {
            cout << coins;
            return 0;
        }
    }
}
```

## 方法二：单层循环写法

这种写法不再用内层循环模拟“这一组有几天”，而是用变量记录当前状态：

```cpp
int dg = 1, coins = 0; // day group，当前每天发多少金币
int di = 0;            // day in group，当前组已经过了几天

for (int i = 1; i <= k; i++) {
    coins += dg;
    di++;

    if (di == dg) {
        dg++;
        di = 0;
    }
}
```

这里可以这样理解：

- `dg` 表示当前处于第几组，也表示今天能拿多少金币。
- `di` 表示当前这一组已经持续了多少天。
- 每过一天，先加上今天的金币 `coins += dg`，再让组内天数 `di++`。
- 如果 `di == dg`，说明当前这一组已经发满了，例如第 3 组已经连续发了 3 天，就进入下一组：`dg++`，并把 `di` 重置为 `0`。

这个方法比嵌套循环更像“状态维护”：每天只循环一次，但要维护好当前金币组和组内计数。

## 方法三：按整组求和写法

这个写法不是一天一天模拟，而是按“整组”来算：

```cpp
int sum = 0, day = 0;

for (int i = 1; day < k; i++) {
    if (day + i <= k) {
        sum += i * i;
    } else {
        sum += i * (k - day);
    }
    day += i;
}
```

这里的 `i` 仍然表示当前组：第 `i` 组有 `i` 天，每天发 `i` 枚金币。

如果 `day + i <= k`，说明当前这一整组都在前 `k` 天内，可以直接加：

```cpp
sum += i * i;
```

因为这一组有 `i` 天，每天 `i` 枚金币，所以总共是 `i * i`。

如果 `day + i > k`，说明最后这一组只能算一部分天数，剩余天数是：

```cpp
k - day
```

所以只加：

```cpp
sum += i * (k - day);
```

你的写法里循环条件是 `day <= k`，结果通常也能算对，因为当 `day == k` 时会多循环一次但加的是 `0`。不过更推荐写成：

```cpp
for (int i = 1; day < k; i++)
```

这样含义更准确：只要已经统计的天数还没到 `k`，就继续算下一组。
