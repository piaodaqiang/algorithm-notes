
## 题型
循环结构 / 级数累加 / 浮点数计算 / 最小值查找

## 本题练习的 C++ 知识
- 用循环不断累加 `1 / n`。
- 用 `double` 保存小数累加结果，并且必须初始化。
- 用 `static_cast<double>(1) / n` 避免整数除法。
- 理解 `while` 和 `for` 中循环变量 `n` 的变化时机。

## 解题思路
题目要求找到最小的 `n`，使得：

```text
S_n = 1 + 1/2 + 1/3 + ... + 1/n > k
```

所以做法就是从 `1/1` 开始，一项一项往后累加。

每加一项，都判断当前和 `S` 是否已经大于 `k`。当第一次出现 `S > k` 时，当前的 `n` 就是答案。因为我们是按照 `1, 2, 3, ...` 的顺序递增尝试的，所以第一次满足条件的 `n` 一定是最小的。

## 方法一：while 循环写法

```cpp
#include<iostream>
using namespace std;

int main()
{
    int k;
    cin >> k;

    double S = 0;
    int n = 0;

    while (S <= k) {
        n++;
        S += static_cast<double>(1) / n;
    }

    cout << n;

    return 0;
}
```

这个写法比较适合初学者理解：

```text
只要 S 还没有大于 k，就继续加下一项。
```

这里 `n` 一开始是 `0`，进入循环后先 `n++`，变成 `1`，再加 `1/1`。下一轮再变成 `2`，加 `1/2`。

## 方法二：for 循环写法

```cpp
#include<iostream>
using namespace std;

int main()
{
    int k;
    cin >> k;

    double S = 0;
    int n;

    for (n = 1; S <= k; n++) {
        S += static_cast<double>(1) / n;
    }

    cout << n - 1;

    return 0;
}
```

`for` 循环的执行顺序是：

```text
初始化 n = 1 -> 判断 S <= k -> 执行循环体 -> n++ -> 再判断
```

所以当循环结束时，`n` 已经多执行了一次 `n++`，比真正答案大 `1`，因此最后要输出：

```cpp
cout << n - 1;
```

## 我容易错的地方

### 1. while 中 `n++` 和累加顺序写反

错误写法：

```cpp
S += static_cast<double>(1) / n;
n++;
```

如果一开始已经有：

```cpp
double S = 1;
int n = 1;
```

这说明 `1/1` 已经算进 `S` 里了。下一次循环应该先让 `n` 变成 `2`，再加 `1/2`。

如果先写：

```cpp
S += static_cast<double>(1) / n;
```

此时 `n` 还是 `1`，就会把 `1/1` 又加一遍，逻辑就错了。

### 2. `double S;` 没有初始化

错误写法：

```cpp
double S;
```

局部变量没有初始化时，里面可能是随机值。第一次判断：

```cpp
S <= k
```

结果就不可靠。正确写法应该是：

```cpp
double S = 0;
```

累加变量一定要有明确的起点。

### 3. for 循环最后要输出 `n - 1`

在 `for (n = 1; S <= k; n++)` 里，循环体执行完以后会先执行一次 `n++`，然后才再次判断条件。

当 `S` 已经大于 `k`，循环停止时，`n` 已经比真正答案多 `1`，所以要输出：

```cpp
cout << n - 1;
```

## 两种写法对比

| 对比点 | while 写法 | for 写法 |
|---|---|---|
| 初始值 | `S = 0, n = 0` | `S = 0, n` 在 `for` 中从 `1` 开始 |
| 每轮什么时候 `n++` | 循环体里先 `n++`，再累加 | 每轮循环体结束后自动 `n++` |
| 最后输出 | `cout << n` | `cout << n - 1` |
| 初学者推荐 | 更推荐，顺序更直观 | 可以掌握，但更容易被 `n++` 绕晕 |
| 容易错的地方 | `n++` 和累加顺序写反 | 忘记最后输出 `n - 1` |

## 关键代码

while 版本核心：

```cpp
double S = 0;
int n = 0;

while (S <= k) {
    n++;
    S += static_cast<double>(1) / n;
}
```

for 版本核心：

```cpp
double S = 0;
int n;

for (n = 1; S <= k; n++) {
    S += static_cast<double>(1) / n;
}

cout << n - 1;
```

## 本题收获
- 累加变量必须初始化，例如 `double S = 0;`。
- 写循环时要特别注意循环变量 `n` 是先变化，还是后变化。
- `for` 循环结束后，循环变量可能已经比答案多变化了一次。
- `static_cast<double>(1) / n` 是为了让除法变成小数除法，避免 `1 / n` 被当成整数除法。
