# algorithm-notes

这是我的长期算法与计算机基础知识库。

本仓库不是临时笔记，而是一个持续维护的个人 CS Wiki，用于沉淀：

- C++ 基础与工程习惯
- STL 常用容器与算法
- 数据结构
- 算法专题
- LeetCode 分类刷题
- 错题复盘
- 算法模板
- 考研机试重点
- 学习记录与工程化总结

## 使用方式

推荐工作流：

```text
Obsidian 编辑 Markdown
        ↓
Git 本地版本管理
        ↓
GitHub 远程备份与展示
```

## 目录说明

| 目录 | 用途 |
| --- | --- |
| `00-系统/` | 知识库规则、路线图、命名规范、标签体系 |
| `C++/` | C++ 语法、语言特性、工程习惯 |
| `STL/` | 容器、迭代器、算法、常用函数 |
| `数据结构/` | 线性表、树、图、堆、并查集等 |
| `算法/` | 搜索、动态规划、贪心、图论、数学等专题 |
| `LeetCode/` | 按题型和专题整理刷题记录 |
| `错题/` | 错题复盘、易错点、反例与修正 |
| `模板/` | 可复用代码模板与解题框架 |
| `考研机试/` | 机试题型、输入输出、常见套路 |

## 长期维护原则

1. 一篇笔记只解决一个清晰主题。
2. 文件名要稳定、可搜索、可长期引用。
3. 重要知识点要能从多个入口找到：目录、标签、反向链接。
4. 错题复盘必须写清楚错误原因，而不是只贴正确代码。
5. 模板代码必须注明适用场景、复杂度和常见坑。
6. 每次提交 Git 都写清楚本次新增或修改了什么。
